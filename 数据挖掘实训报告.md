## 摘要

本实训按照CRISP-DM标准流程完成“业务理解→数据理解→数据准备→建模→评估→部署”全链条工作。前期对本地服装目录共11287张、3个粗粒度子类进行量化探索，并与DeepFashion官方基准20000张、50个细类、规范标注对比，最终选定后者作为唯一训练数据源，**据此将任务界定为DeepFashion所定义的50类服饰图像多分类**。

数据准备阶段通过自定义PyTorch Dataset完成标注解析、路径修复、数据增强与类别分布统计；建模阶段采用ImageNet预训练ResNet18进行**全网络微调**，卷积骨干未冻结，并配合标签平滑、AdamW、余弦退火学习率与Early Stopping。优化后模型在验证集Epoch 22取得最佳Top-1准确率61.55%，独立评估脚本复现为61.50%；Top-3准确率为79.75%。独立测试集Top-1为58.73%，Top-3为79.00%。Android端推理已与训练预处理对齐，采用Resize 256与CenterCrop 224，消除Training-Serving Skew。

本报告重点给出各环节方法原理、核心代码与实测评估结果，含混淆矩阵与错例分析；Android应用作为模型部署验证，不展开界面细节。

## 第1章 课题背景与数据挖掘流程

### 1.1 研究背景

服饰图像自动分类在电商标注、库存管理与衣橱整理中具有业务价值。将图像映射到预定义类别，属于监督式多分类数据挖掘任务：输入为高维像素特征，输出为类别标签与置信度，可用于辅助决策与后续统计分析。

### 1.2 任务定义

给定样本集 D={(x_i, y_i)}，x_i 为RGB服饰图像，y_i 为50类标签之一，编号0～49，学习分类模型 f(x) 使未见样本上的分类性能最优。工程约束为：模型需可导出为ONNX并在Android端离线推理。

### 1.3 CRISP-DM流程映射

## 第4章 算法原理与模型构建

### 4.1 传统方法与深度方法对比

**表5 传统方法与深度学习方法对比**

| 方法 | 特征 | 分类器 | 课设是否实验 |
|------|------|--------|--------------|
| 颜色直方图+SVM | 手工 | SVM | 未单独实现，细粒度效果预期较差 |
| 从零训练ResNet18 | 自动 | Softmax | 未采用，收敛慢且易过拟合 |
| **ImageNet预训练+全网络微调** | 自动 | Softmax | **最终方案** |

课设选型理由：在DeepFashion 50类细粒度设定下，视觉差异需层次化特征；该数据集规模不足以从零训练深网；迁移学习+全网络微调在精度与训练成本间折中最优。

### 4.2 迁移学习与全网络微调

本实训**未冻结**卷积骨干，而是加载ImageNet预训练权重后，**替换分类头并对全部参数进行微调**，全部网络参数纳入AdamW优化。这与“仅训练最后一层”不同：骨干低层保留通用边缘/纹理特征，高层随服饰域数据继续调整，以适配细粒度差异。

**图7** ResNet18 全网络微调结构示意

图7中，反向传播路径覆盖骨干与分类头；Dropout(0.6/0.5)仅在分类头，用于抑制过拟合。

```python
def get_model():
    model = resnet18(weights=ResNet18_Weights.IMAGENET1K_V1)
    num_features = model.fc.in_features
    model.fc = nn.Sequential(
        nn.Dropout(0.6),
        nn.Linear(num_features, 512),
        nn.ReLU(inplace=True),
        nn.Dropout(0.5),
        nn.Linear(512, 50)
    )
    return model  # 全部参数参与 optimizer
```

### 4.3 损失函数与优化

```python
criterion = nn.CrossEntropyLoss(label_smoothing=0.1)
optimizer = optim.AdamW(model.parameters(), lr=0.001, weight_decay=2e-4)
scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=remaining_epochs, eta_min=1e-6)
```

标签平滑将硬标签软化为(1-ε)·onehot + ε/K，减轻过度自信；AdamW解耦权重衰减；余弦退火使后期学习率趋近1e-6，精细收敛。

### 4.4 训练/验证循环

```python
def validate(model, dataloader, criterion, device, scaler=None):
    model.eval()
    running_loss = 0.0
    correct = 0
    total = 0
    with torch.no_grad():
        for images, labels in dataloader:
            images, labels = images.to(device), labels.to(device)
            outputs = model(images)
            loss = criterion(outputs, labels)
            running_loss += loss.item()
            _, predicted = outputs.max(1)
            total += labels.size(0)
            correct += predicted.eq(labels).sum().item()
    epoch_loss = running_loss / len(dataloader)
    epoch_acc = 100.0 * correct / total
    return epoch_loss, epoch_acc
```

model.eval()关闭Dropout；torch.no_grad()节省显存；Top-1准确率由correct/total直接统计，与第6章评估脚本口径一致。

### 5.3 Early Stopping与checkpoint

```python
if val_acc > best_val_acc:
    best_val_acc = val_acc
    no_improve_epochs = 0
    torch.save(model.state_dict(), best_model_path)
else:
    no_improve_epochs += 1
if no_improve_epochs >= early_stop_patience:
    break
```

Early Stopping是模型选择标准技术：以验证集为准保存最优权重，而非最后一轮。

## 第7章 模型部署与应用转化

### 7.1 部署链路与训练—服务一致性

Android端采用与训练端一致的Resize 256与CenterCrop 224预处理，在DeepFashionClassifier中实现，以避免Training-Serving Skew：

```kotlin
private fun preprocessForInference(source: Bitmap): Bitmap {
    val scaled = Bitmap.createScaledBitmap(source, 256, 256, true)
    val offset = (256 - 224) / 2
    return Bitmap.createBitmap(scaled, offset, offset, 224, 224)
}
```

归一化仍使用ImageNet均值方差，与Python端Normalize一致。移动端推理与验证集评估因此采用同一几何变换。

## 第8章 总结与展望

### 8.1 工作总结

本实训按 CRISP-DM 完成数据挖掘闭环：量化评估本地 11287 张粗类数据后选用 DeepFashion；预处理含标注对齐与增强；ResNet18全网络微调取得验证Top-1 61.55%、复现61.50%、测试58.73%；混淆矩阵揭示 Dress 过预测及 Skirt/Top 等易混对；ONNX 在真实样本上与 PyTorch 一致。

### 8.2 挖掘结论（知识输出）

**数据决策**：本地11287张粗类数据不可直接用于DeepFashion所定义的50类分类任务，故选定DeepFashion为唯一训练源。

**模型规律**：Dress样本最多，模型对其有过预测倾向；Skirt、Top、Blouse等细类是主要混淆源。

**部署要点**：预处理必须与训练一致；ONNX在真实样本上与PyTorch logits差小于1e-5。

### 8.3 展望

类别重采样/Focal Loss缓解Dress过预测；针对表12类对做难例挖掘；EfficientNet骨干；本地服装目录补充50类标注后做域适应。
